# LLM Wiki Schema — 개인 지식 베이스 운영 지침

> **LLM에게:** 이 파일은 이 Obsidian 보관함을 유지·관리하는 방법을 정의합니다.
> 새 세션을 시작할 때 반드시 이 파일을 먼저 읽고 `index.md`를 확인하세요.

---

## 보관함 개요

**목적:** 투자 회고, 독서록, 개발 기록, 개인 기록을 LLM이 유지하는 위키로 구조화한 개인 지식 베이스.

**운영 원칙:**
- `raw/` 는 불변 소스 문서 — LLM이 읽기만 하고 수정하지 않음
- `wiki/` 는 LLM이 작성·유지하는 합성 지식 — 사용자가 직접 수정하지 않아도 됨
- `templates/` 는 새 문서 작성 시 참조하는 양식
- `Home.md` 의 Dataview가 목록·통계를 자동 생성 — `index.md` 에는 핵심 페이지만 큐레이션
- 모든 작업 후 `index.md`(핵심 페이지만)와 `log.md`를 업데이트

**설치된 플러그인:**
- **Dataview** — Home.md 대시보드 자동 생성 (목록·통계). 이 보관함의 핵심 자동화 도구.
- **Omnisearch** — 전체 텍스트 검색
- **Folder Notes** — 폴더를 노트로 표현
- **Tasks** — 작업 항목 추적

## 디렉토리 구조

```
obsidian/                          ← 보관함 루트
├── CLAUDE.md                      ← 이 파일 (LLM 운영 지침)
├── Home.md                        ← 보관함 홈/대시보드 (Dataview 자동)
├── index.md                       ← 핵심 wiki 페이지 큐레이션 (LLM이 유지)
├── log.md                         ← 작업 이력 로그 (append-only)
│
├── raw/                           ← 원본 소스 문서 (불변)
│   ├── investments/               ← 투자 관련 기사, 리포트, 클리핑
│   ├── books/                     ← 도서 클리핑, 챕터 노트
│   ├── dev/                       ← 개발 아티클, 공식 문서 발췌
│   ├── personal/                  ← 개인 일지, 메모 원문
│   └── assets/                    ← 이미지, 첨부파일 (Obsidian 첨부 폴더)
│
├── wiki/                          ← LLM이 작성·유지하는 합성 지식
│   ├── investments/
│   │   ├── 종목/                  ← 개별 종목 분석 페이지
│   │   ├── 섹터/                  ← 섹터/업종 분석
│   │   ├── 회고/                  ← 월간·분기 투자 회고 + 매매일지
│   │   └── 개념/                  ← 투자 개념, 전략, 용어
│   ├── books/
│   │   ├── 독서록/                ← 개별 도서 요약·분석
│   │   └── 개념/                  ← 도서에서 추출한 핵심 개념
│   ├── dev/
│   │   ├── 프로젝트/              ← 개발 프로젝트 기록
│   │   ├── 기술/                  ← 기술·도구·언어 페이지
│   │   └── TIL/                   ← Today I Learned (날짜별)
│   └── personal/
│       ├── 목표/                  ← 목표 설정·추적
│       ├── 회고/                  ← 개인 회고 (주간·월간)
│       └── 아이디어/              ← 아이디어, 인사이트
│
└── templates/                     ← Obsidian 템플릿
    ├── 매매일지.md
    ├── 투자회고.md
    ├── 독서록.md
    ├── TIL.md
    ├── 개인회고.md
    └── 종목분석.md
```

## 파일 명명 규칙

| 영역 | 형식 | 예시 |
|------|------|------|
| 매매일지 | `YYYYMMDD_종목명_매수\|매도.md` | `20260617_삼성전자_매수.md` |
| 투자회고 | `YYYY-MM_투자회고.md` | `2026-06_투자회고.md` |
| 투자복기 | `YYYYMMDD_종목명_복기.md` | `20260627_TIGER미국S&P500_복기.md` |
| 독서록 | `저자명_책제목.md` | `찰리멍거_가난한찰리의연감.md` |
| TIL | `YYYYMMDD_주제.md` | `20260617_React_useCallback.md` |
| 종목 분석 | `종목명.md` | `삼성전자.md` |
| 개인회고 | `YYYY-WNN_주간회고.md` 또는 `YYYY-MM_월간회고.md` | `2026-W25_주간회고.md` |
| 아이디어 | `아이디어_제목.md` | `아이디어_자동매매시스템구상.md` |

> **파일명 고유성 유지:** 짧은 이름 링크를 사용하므로 보관함 전체에서 파일명이 겹치지 않도록 합니다.
> 예: 독서 개념과 투자 개념이 같은 이름이면 앞에 구분어를 붙이세요 (`안전마진-투자.md` vs `안전마진-독서.md`).

## Frontmatter 규칙

### 매매일지
```yaml
---
type: 매매일지
date: YYYY-MM-DD
ticker: 005930
종목: 삼성전자
sector: 반도체
거래유형: 매수        # 매수 | 매도 | 분할매수 | 분할매도
전략: 중장기보유      # 단기트레이딩 | 중장기보유
체결가: 75000
수량: 10
수익률:              # 매도 후 기입 (%)
status: 보유중        # 보유중 | 청산 | 복기완료
tags: [매매일지, 삼성전자, 반도체]
---
```

### 투자 관련 (회고·종목·섹터·개념)
```yaml
---
type: 투자회고 | 종목분석 | 섹터분석 | 투자개념
date: YYYY-MM-DD
ticker: 삼성전자 | AAPL
sector: IT | 반도체 | 금융
tags: [투자, 종목명, 섹터명]
status: 진행중 | 완료 | 복기완료
---
```

### 독서 관련
```yaml
---
type: 독서록 | 독서개념
date: YYYY-MM-DD
author: 저자명
title: 책제목
genre: 투자 | 자기계발 | 기술 | 철학 | 소설
rating: 1-5
status: 읽는중 | 완독 | 중단
tags: [독서, 장르, 저자명]
---
```

### 개발 관련
```yaml
---
type: TIL | 프로젝트 | 기술노트
date: YYYY-MM-DD
tech: React | Python | Go | 등
tags: [개발, 기술명]
---
```

### 개인 관련


## 단축 트리거

### 파일명만 입력 → 자동 인제스트

사용자가 **파일명만 단독으로 입력**하면 (예: `삼성전자.md`, `찰리멍거_책`, `20260626_TIL`) 다음 절차를 자동 실행한다.

1. `raw/` 하위 디렉토리 전체에서 해당 파일을 재귀 탐색
2. 파일을 찾으면 **인제스트 절차(아래 §1)** 를 즉시 실행
3. 파일을 찾지 못하면 "찾을 수 없습니다. 전체 경로를 알려주세요." 라고만 응답

**판별 기준:** 메시지가 경로 구분자(`/`)나 동사·조사를 포함하지 않고 파일명처럼 보이면 이 트리거로 처리한다.  
확장자(`.md`)는 생략해도 된다.

## LLM 운영 절차

### 1. 인제스트 (새 소스 추가)

사용자가 새 소스를 `raw/`에 추가하면:

1. 소스 읽기 및 핵심 내용 파악
2. 사용자와 주요 인사이트 논의 (필요시)
3. `wiki/` 해당 폴더에 요약·분석 페이지 작성 (frontmatter 반드시 포함)
4. 연관된 기존 wiki 페이지 업데이트 (종목, 개념, 섹터 등)
5. 중요 페이지면 `index.md`에 한 줄 설명과 함께 추가 (전수 기록 불필요)
6. `log.md`에 `## [YYYY-MM-DD] ingest | 제목` 형식으로 기록

### 2. 쿼리 (질문에 답하기)

1. `index.md`를 읽어 관련 페이지 파악
2. 관련 wiki 페이지들 읽기
3. 답변 합성 및 출처(페이지명) 명시
4. 가치 있는 답변이면 wiki 페이지로 저장 제안
5. `log.md`에 `## [YYYY-MM-DD] query | 질문요약` 형식으로 기록

### 3. 린트 (wiki 건강 검진)

주기적으로 다음을 점검:
- 모순되는 정보가 있는 페이지 쌍
- 신규 소스로 인해 오래된 주장이 된 내용
- 인바운드 링크가 없는 고아 페이지
- 언급되지만 별도 페이지가 없는 중요 개념
- 빠진 교차 참조
- `log.md`에 `## [YYYY-MM-DD] lint | 결과요약` 형식으로 기록

## 링크 규칙

- **짧은 이름 사용:** `[[페이지명]]` 형식으로 파일명만 사용 (풀패스 사용 금지)
- **별칭 사용:** `[[페이지명|표시텍스트]]`
- **파일명 고유성 유지** — 이름이 겹치면 앞에 영역 구분어 추가
- 독서록은 핵심 개념 페이지와 양방향 연결
- TIL은 관련 기술 페이지와 연결
- 종목 매매일지는 종목 분석 페이지와 연결

## 투자 wiki 추가 지침

- 종목 페이지에는 매매일지 링크, 섹터 링크, 핵심 지표 포함
- 회고 페이지에는 해당 기간 매매일지 전체 링크
- 새로운 투자 개념 발견 시 `wiki/investments/개념/`에 별도 페이지 생성
- 섹터 분석은 관련 종목 페이지 모두 연결

## 독서 wiki 추가 지침

- 독서록 페이지: 1페이지 요약 + 핵심 인용 + 적용 아이디어
- 여러 책에서 공통으로 나오는 개념은 `wiki/books/개념/`으로 통합
- 투자 서적은 `wiki/investments/개념/`과도 교차 연결

### 문학 vs 비문학 처리 방침

**이 독서록은 시험 대비가 아닌 순수한 문학적 소양 향상을 위한 것이다.** 장르에 따라 접근을 다르게 한다.

#### 문학 작품 (소설·시·에세이·희곡 등)

"정답"을 제시하지 않는다. 대신:

- **작가 이력**: 생애·시대적 배경·주요 작품 흐름 — 작품을 낳은 맥락 제공
- **작품 평가**: 문학사적 위치, 비평가·독자 반응, 이 작품이 왜 중요한지
- **독해 가이드**: 주제·상징·구조를 독자가 스스로 발견하도록 질문 형태로 안내
- **감상 심화**: 사용자의 감상을 먼저 듣고, 거기에 새로운 시각이나 관련 작품을 연결해 확장

> 목표: 사용자가 작품을 더 깊이·풍부하게 경험하도록 돕는 것. 해석을 주입하지 않는다.

#### 비문학 (자기계발·경영·투자·과학·역사 등)

- **핵심 주장 정리**: 저자가 말하는 것을 명확하게 요약
- **비평**: 논거의 강점·약점, 반론, 실증 근거 평가
- **설명 보완**: 사용자가 이해하기 어려운 개념·용어를 풀어서 설명
- **적용 아이디어**: 사용자의 상황(투자·개발·개인)에 연결되는 실천 포인트 제안

### 다회독 처리 방침

사용자가 이미 wiki 페이지가 존재하는 책을 다시 가져오면 **새 페이지를 만들지 않고 기존 페이지를 업데이트**한다.

1. `read_count` +1, `last_read` 갱신, `회독 기록` 테이블에 새 행 추가
2. 기존 페이지 끝에 `## N독 — YYYY-MM-DD` 섹션 추가 (템플릿의 N독 블록 사용)
3. **이전 회독과의 차이**에 집중: 새로 보인 것, 시각의 변화, 새 문장 발췌
4. 2독 이상이 되면 `## 종합 평가` 섹션을 작성하거나 업데이트
5. `log.md`에 `## [YYYY-MM-DD] reread N독 | 책제목` 형식으로 기록

> 같은 책을 여러 번 읽을수록 wiki 페이지가 깊어지는 구조다. 덮어쓰지 말고 쌓아간다.

## 개발 wiki 추가 지침

- TIL은 날짜별 단일 항목, 관련 기술 페이지에 역링크
- 프로젝트 페이지: 목표, 기술 스택, 진행 상황, 회고 섹션 포함
- 기술 페이지: 공식 문서 링크, 개인 사용 경험, 관련 TIL 목록

## 중요 메모

- 이 보관함은 사용 초기 단계입니다. 문서가 추가될수록 wiki가 성장합니다.
- wiki 페이지 작성 시 frontmatter(type, date, tags 등)를 반드시 포함하세요 — Dataview 자동 집계에 사용됩니다.
- `Home.md`의 Dataview가 목록·통계를 자동 생성합니다. `index.md`에는 중요 페이지만 수동 큐레이션하세요.
- Obsidian 첨부 파일 경로는 `raw/assets/`로 설정하세요.
