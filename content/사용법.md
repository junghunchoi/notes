# 보관함 사용법

> LLM(Claude Code)과 함께 운영하는 개인 지식 베이스입니다.
> LLM이 wiki를 작성·유지하고, 사람은 소스를 넣고 질문합니다.

---

## 기본 개념

이 보관함은 세 레이어로 구성됩니다.

| 레이어 | 폴더 | 역할 | 누가 쓰나 |
|--------|------|------|-----------|
| 원본 소스 | `raw/` | 기사, 리포트, 메모 원문 | 사람 |
| 위키 | `wiki/` | 요약, 분석, 연결된 지식 | LLM |
| 템플릿 | `templates/` | 문서 작성 양식 | 둘 다 |

**Home.md 대시보드:** Dataview 플러그인이 자동으로 보유 종목, 최근 매매, 독서 현황, 최근 TIL을 집계합니다.
wiki 페이지를 작성할 때 frontmatter(type, date 등)를 채우면 대시보드에 자동 반영됩니다.

---

## 자주 하는 작업

### 1. 새 소스 추가 (인제스트)

1. 정리할 내용을 `raw/` 하위 폴더에 저장합니다.
   - 투자 관련 → `raw/investments/`
   - 책/독서 → `raw/books/`
   - 개발 자료 → `raw/dev/`
   - 개인 메모 → `raw/personal/`
2. Claude Code에 요청합니다.

```
raw/investments/삼성전자_실적발표.md 처리해줘
```

LLM이 알아서 wiki 페이지를 만들고, index와 log를 업데이트합니다.

---

### 2. 매매일지 작성

1. `templates/매매일지.md`를 복사합니다.
2. 파일명: `YYYYMMDD_종목명_매수.md` (예: `20260618_삼성전자_매수.md`)
3. `wiki/investments/회고/` 또는 `wiki/investments/종목/` 아래에 저장합니다.
4. **frontmatter 필드를 채웁니다.** — Home.md 대시보드에 자동 반영됩니다.

```yaml
---
type: 매매일지
date: 2026-06-18
ticker: 005930
종목: 삼성전자
sector: 반도체
거래유형: 매수
전략: 중장기보유
체결가: 75000
수량: 10
수익률:          ← 매도 후 기입
status: 보유중
tags: [매매일지, 삼성전자]
---
```

5. 작성 후 LLM에게 종목 페이지 업데이트를 요청합니다.

```
20260618_삼성전자_매수.md 작성했어. 삼성전자 종목 페이지에 반영해줘.
```

---

### 3. 독서록 작성

1. `templates/독서록.md`를 복사합니다.
2. 파일명: `저자명_책제목.md`
3. `wiki/books/독서록/` 아래에 저장합니다.
4. 책을 다 읽으면 LLM에게 핵심 개념 페이지 생성을 요청합니다.

```
찰리멍거_가난한찰리의연감.md 완독했어. 핵심 개념 정리해줘.
```

---

### 4. TIL (개발 메모) 작성

1. `templates/TIL.md`를 복사합니다.
2. 파일명: `YYYYMMDD_주제.md`
3. `wiki/dev/TIL/` 아래에 저장합니다.

---

### 5. 투자 회고 요청

월말이나 분기말에 LLM에게 요청합니다.

```
2026년 6월 투자 회고 작성해줘.
```

LLM이 해당 기간의 매매일지를 참고해 회고 페이지를 만들어 줍니다.

---

### 6. 질문하기

wiki에 쌓인 내용을 바탕으로 자유롭게 질문합니다.

```
내가 가장 많이 손실 본 패턴이 뭐야?
최근에 읽은 투자 책에서 공통으로 강조한 원칙이 있어?
이번 달 개발 TIL 요약해줘.
```

---

### 7. Wiki 점검 (린트)

주기적으로 wiki 상태를 점검합니다.

```
wiki 린트 해줘.
```

LLM이 고아 페이지, 오래된 정보, 빠진 연결 등을 점검하고 보고합니다.

## 폴더 구조 한눈에 보기

```
obsidian/
├── 사용법.md          ← 이 파일
├── Home.md            ← 대시보드 (Dataview 자동 집계)
├── index.md           ← 핵심 wiki 페이지 큐레이션 (LLM이 유지)
├── log.md             ← 작업 이력 (LLM이 유지)
├── CLAUDE.md          ← LLM 운영 지침 (건드리지 않아도 됨)
├── 포트폴리오.base    ← Bases 뷰 (Obsidian에서 열면 종목 테이블)
│
├── raw/               ← 여기에 원본 파일을 넣으세요
│   ├── investments/
│   ├── books/
│   ├── dev/
│   ├── personal/
│   └── assets/        ← 이미지 자동 저장 폴더
│
├── wiki/              ← LLM이 자동으로 채워줍니다
│   ├── investments/   (종목 / 섹터 / 회고 / 개념)
│   ├── books/         (독서록 / 개념)
│   ├── dev/           (TIL / 프로젝트 / 기술)
│   └── personal/      (목표 / 회고 / 아이디어)
│
└── templates/         ← 문서 작성 시 복사해서 사용
    ├── 매매일지.md
    ├── 투자회고.md
    ├── 독서록.md
    ├── TIL.md
    ├── 개인회고.md
    └── 종목분석.md
```

## 링크 표기 원칙

보관함 내 링크는 **짧은 이름(파일명만)** 을 사용합니다.

```markdown
✅ 올바름: [[삼성전자]], [[2026-06_투자회고]], [[가난한찰리의연감]]
❌ 지양:   [[wiki/investments/종목/삼성전자]]
```

Obsidian이 파일명으로 자동 연결하므로 풀패스 불필요. 단, 파일명이 겹치지 않도록 고유하게 지어주세요.

## Obsidian 설정

| 설정 | 위치 | 값 |
|------|------|-----|
| 첨부 파일 폴더 | Settings → Files and links → Attachment folder path | `raw/assets` |
| 템플릿 폴더 | Settings → Core plugins → Templates → Template folder location | `templates` |
| 새 노트 위치 | Settings → Files and links → Default location for new notes | `raw` |

**설치된 플러그인:**

| 플러그인 | 역할 |
|----------|------|
| **Dataview** | Home.md 대시보드 자동 집계 — 핵심 자동화 도구 |
| **Omnisearch** | 전체 텍스트 검색 |
| **Folder Notes** | 폴더를 노트로 표현 |
| **Tasks** | 체크리스트·작업 추적 |

> **선택 설치:** Obsidian Git (자동 백업/버전관리 원할 때)

---

## 핵심 원칙

- `raw/`는 원본 보존 — 수정하지 마세요.
- `wiki/`는 LLM이 쓰는 공간 — 직접 수정하지 않아도 됩니다.
- wiki 페이지 작성 시 frontmatter를 반드시 채우세요 — Home 대시보드에 자동 반영됩니다.
- 새 문서를 넣고 LLM에게 말 걸면 나머지는 자동입니다.
